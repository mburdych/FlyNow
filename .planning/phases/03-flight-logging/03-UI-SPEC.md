## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tight gap between inline input labels and helper text |
| sm | 8px | Vertical spacing inside each form field group |
| md | 16px | Default in-card spacing, input padding rhythm, list row gap |
| lg | 24px | Separation between log form section and flight history section |
| xl | 32px | Major split between status area and logging area in the card |
| 2xl | 48px | Dashboard stack spacing when this card is grouped with others |
| 3xl | 64px | Reserved for page-level spacing only (not inside card body) |

Exceptions: Minimum interactive target height is 48px for all form controls (selects, date/time picker, submit action), even where visual padding suggests smaller size.

## Form And Interaction Contract

### Field Contract (LOG-01)

| Field | Input Type | Required | Default | Validation / Rules |
|------|------------|----------|---------|--------------------|
| Date | Date picker | Yes | Today (local date) | Must be valid local date; cannot be empty |
| Balloon | Select | Yes | Last used balloon | Allowed values: `OM-0007`, `OM-0008` |
| Launch time | Local 24h time picker | Yes | Empty (explicit selection required) | Format `HH:mm`, local time, no free-text input |
| Duration | Number input (minutes) or select | Yes | 90 | Positive integer minutes; prefilled 90 |
| Site | Select | Yes | Selected site from card context | Must map to known site identifier/name in card data |
| Outcome | Select | Yes | `flown` | Allowed values: `flown`, `cancelled-weather`, `cancelled-other` |
| Notes | Textarea | No | Empty | Optional free text; preserve exactly as entered |

### Submission Behavior

- Primary focal point is the `Log Flight` CTA plus the inline submission status region directly below it.
- Disable submit while request is in-flight to prevent duplicate writes.
- On successful save, keep date/balloon/site defaults sticky (today/last used/selected site), reset launch time and notes.
- Show inline success feedback within 2 seconds: `Flight logged locally`.
- On failure, preserve all current form values and show error copy from Copywriting Contract.

### History List Contract

- Render newest-first list of previously logged flights.
- Each row must show: date, launch time, balloon, site, duration, outcome, and notes preview (if present).
- If notes are absent, show `No notes` in neutral body style.
- Empty list state must use the empty state copy contract verbatim.
