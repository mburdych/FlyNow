## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline chip padding/gaps |
| sm | 8px | Compact row spacing, form input padding |
| md | 16px | Card internal default spacing |
| lg | 24px | Larger section rhythm in settings/sections |
| xl | 32px | Major intra-card separation when needed |
| 2xl | 48px | Large section breaks (rare in card scope) |
| 3xl | 64px | Not expected in this phase; reserved token |

Exceptions: segmented control tap targets MAY use 44px minimum height for touch accessibility.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 | 1.5 |
| Label | 12px | 600 | 1.4 |
| Heading | 14px | 600 | 1.2 |
| Display | 20px | 600 | 1.2 |

Contract constraints:
- Only two weights are allowed in this phase: 400 and 600.
- Language toggle labels (`SK`, `EN`) use Label role and selected state emphasis via color/background, not heavier weight.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--ha-card-background, var(--card-background-color))` | Card background and panel surfaces |
| Secondary (30%) | `var(--secondary-background-color, rgba(127,127,127,0.12))` | Segmented control track, collapsible settings container, subtle grouped areas |
| Accent (10%) | `var(--primary-color)` | Active language segment, focus ring, selected border emphasis |
| Destructive | `var(--error-color)` | Existing destructive/error-only messaging states |

Accent reserved for: active `SK/EN` segment, keyboard focus treatment on toggle controls, and selected-state border in settings row only.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Switch language |
| Empty state heading | Language preference not set |
| Empty state body | We selected a language from Home Assistant on first load. Use SK/EN to change it anytime. |
| Error state | Could not save language preference. Try switching again or reload the dashboard. |
| Destructive confirmation | none (no destructive action in this phase) |

Localization contract:
- Every new and existing visible card string must be dictionary-backed with strict SK/EN key parity (D-07, D-08).
- Missing localized key at runtime must render English fallback for that key and continue rendering (D-09).

---

## Interaction Contract

1. Language selector is exposed as explicit `SK/EN` segmented control inside a collapsible settings panel in card header context (D-01, D-03).
2. Toggle interaction is one-tap direct switch between `sk` and `en` (D-02), with immediate rerender of all card labels/messages.
3. Initialization order is strict:
   - Read dedicated `localStorage` key.
   - If value is valid (`sk` or `en`), use it.
   - If missing/invalid, infer once from `hass.language`, persist result immediately, then use persisted value (D-04, D-05, D-06).
4. After first persisted value exists, browser/HA locale changes must not override explicit user selection.
5. Keyboard behavior: segmented control must be reachable with Tab and togglable via Enter/Space.
