## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FlyNow integration -> HA notify/calendar services | Untrusted config-driven entity IDs and message payloads cross into external service calls |
| Coordinator state -> notification formatter | Weather/state attributes become user-visible outbound messages |
| Dedup ledger persistence -> runtime dispatch decision | Stored timestamps directly gate whether alerts are sent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | S | notifier target selection | mitigate | Validate notifier/calendar entity IDs from config flow and only call configured allowlisted IDs |
| T-02-02 | T | outbound message payload | mitigate | Build messages from fixed templates and sanitized numeric fields; no dynamic template execution |
| T-02-03 | R | dedup send accountability | mitigate | Log channel result summaries with window identity and success/failure outcome |
| T-02-04 | I | cross-channel disclosure | mitigate | Route crew/pilot/private targets separately and avoid broadcasting to implicit default notifiers |
| T-02-05 | D | notification spam | mitigate | Enforce D-03/D-04 dedup key and 1-hour cooldown beginning at first successful send |
| T-02-06 | E | unauthorized action invocation | accept | Home Assistant auth model controls service call permissions; no custom privilege layer added in this phase |
</threat_model>
