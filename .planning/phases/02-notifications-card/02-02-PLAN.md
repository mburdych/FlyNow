<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HA entity attributes -> card render model | Untrusted/missing data reaches user-facing HTML output |
| Card config -> selected entity ID | Dashboard configuration selects what state object is rendered |
| Cached state -> stale UI | Last known values can outlive fresh backend state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | S | card entity binding | mitigate | Default to `binary_sensor.flynow_status` and validate selected entity exists before render |
| T-02-08 | T | rendered condition values | mitigate | Coerce numeric fields and fallback safely when attributes are missing or malformed |
| T-02-09 | R | stale badge correctness | mitigate | Render explicit stale marker whenever fallback cache path is used |
| T-02-10 | I | dashboard information exposure | accept | Card shows operational weather status only; no credentials or sensitive payloads rendered |
| T-02-11 | D | UI churn from rapid updates | mitigate | Use Lit reactive update flow without custom polling; avoid repeated timers or loops |
| T-02-12 | E | unsafe script execution in UI | mitigate | Use Lit templating with escaped content and no `unsafeHTML` for sensor-derived values |
</threat_model>
