# Phase 999.1 — Learnings

## Decisions

- **Header segmented SK/EN control over settings row / footer link.** One-tap direct toggle in collapsible header. Source: `999.1-DISCUSSION-LOG.md` (Language Selector UX).
- **localStorage over HA user preference / session-only.** Card-side persistence keeps the toggle independent of HA account state and survives HA restarts. Key: `flynow.card.language`. Source: `999.1-DISCUSSION-LOG.md` (Persistence Policy).
- **Single typed dictionary over per-language files / inline strings.** `Record<LanguageCode, Record<TranslationKey, string>>` gives compile-time key parity at zero runtime cost. Source: `999.1-DISCUSSION-LOG.md` (Translation Architecture) + `999.1-01-PLAN.md` (must_haves.truths).
- **English-as-runtime-fallback when key missing.** Avoids broken render even if a key is added in EN but forgotten in SK. Source: `999.1-DISCUSSION-LOG.md` (Translation Architecture notes).
- **First-load fallback to `hass.language` then persist immediately.** Card never operates in "undecided language" state — first read either inherits or sets a deterministic default. Source: `999.1-01-PLAN.md` (truths).

## Patterns

- **Typed translation dictionary pattern.** `Record<LanguageCode, Record<TranslationKey, string>>` where both `LanguageCode` and `TranslationKey` are exhaustive string-literal unions. The TypeScript compiler enforces key parity across all languages and across all `t(key)` call sites. Reusable for any future HA card or frontend addition. Source: `lovelace/flynow-card/src/types.ts` + `flynow-card.ts` (commit `2bc1c8a`).
- **3-tier i18n init lifecycle for HA cards.** `localStorage[KEY] → if invalid/missing fall back to hass.language → persist immediately`. Survives reload, decouples from HA locale changes after explicit user choice. Reusable for any persisted card-side preference (theme, units, etc.), not just language. Source: `999.1-01-PLAN.md` (truths) + commit `2bc1c8a`.
- **Python+regex as cheap automated validation for TS code without test harness.** PLAN's automated validation pipeline used `python -c "assert 'localStorage' in src and 'hass.language' in src"` as a smoke test instead of standing up a JS test runner. Low setup cost, catches gross structural regressions. Pattern fits one-off frontend phases. Source: `999.1-VALIDATION.md` (Automated Validation Pipeline).
- **Falsifiable PLAN must_haves enable goal-backward verification.** PLAN's `must_haves.truths` + `key_links` (with regex `pattern`) acted as a contract that could be programmatically cross-checked post-execution. Made spot-check trivial after Cursor's commit. Reusable for all future PLANs. Source: `999.1-01-PLAN.md` frontmatter.

Lesson: review the full diff, not just the snippet you're scanning for. Source: `CEILING-FOG-CORRECTIONS.md` revised status table.

Worked first try with checksum verification gating. Pattern documented in this session for future remote deploys when VPN client cannot be installed. Source: this session's deploy negotiation.
- **0x0.st (and most ephemeral file hosts) are now disabled due to AI bot abuse.** Tried `0x0.st`, `bashupload.com`, `catbox.moe` — all blocked or unreachable. Fallback to base64 inline paste was actually the cleanest path. Source: this session's upload attempts (HTTP 503 from 0x0.st).
